## Обзор проекта

**Demo Shop Bot** — это готовая база для Telegram-магазина, которую можно адаптировать под витрину товаров, каталог услуг или небольшой e-commerce проект.

Репозиторий показывает практическую реализацию Telegram-бота на **aiogram 3** с:
- каталогом и корзиной,
- пошаговым оформлением заказа через FSM,
- пользовательским профилем,
- административной частью,
- локализацией на русском и английском языках,
- хранением данных в **SQLite** через **Tortoise ORM**.

Проект хорошо подходит  как стартовая основа для кастомизации под конкретный бизнес-сценарий.

## Что уже реализовано

- каталог товаров с категориями, пагинацией и карточками товаров
- корзина с добавлением, изменением количества и удалением позиций
- пошаговое оформление заказа
- хранение временных состояний оформления в Redis
- пользовательский профиль с редактированием данных
- автозаполнение данных из профиля при оформлении заказа
- создание заказа и сохранение его состава в базе данных
- история заказов пользователя
- административная панель для управления категориями, товарами и заказами
- поиск по товарам и заказам
- статистика по заказам и экспорт заказов в CSV
- поддержка двух языков интерфейса
- миграции через Aerich
- резервное копирование SQLite-базы перед применением миграций

## Текущее состояние проекта

На текущем этапе проект ориентирован на **локальный запуск** и демонстрирует основную логику магазина внутри Telegram.

Что важно учитывать:
- в проекте уже есть оформление заказа и выбор способа оплаты
- онлайн-оплата пока не подключена как рабочий пользовательский сценарий
- инфраструктура для Docker и автоматизированных тестов пока не добавлена
- база данных по умолчанию настроена на SQLite
- Redis используется для хранения состояний во время пользовательских сценариев

## Структура проекта

```text
Demo_Shop_Bot/
├── assets/               # изображения и демонстрационные материалы
├── bot/
│   ├── handlers/         # пользовательские и административные обработчики
│   ├── keyboards/        # клавиатуры бота
│   ├── states/           # состояния пользовательских и админских сценариев
│   └── utils/            # вспомогательная логика
├── config_data/          # конфигурация и инициализация бота
├── database/             # модели, конфигурация БД, CRUD и инициализация
├── migrations/           # миграции Aerich
├── services/             # локализация и сервисный слой
├── logging_config.py     # конфигурация логирования
├── main.py               # точка входа
├── pyproject.toml        # конфигурация Aerich
└── requirements.txt      # зависимости проекта
```

## Основные сущности

- **User** — данные пользователя Telegram
- **UserLocale** — выбранный язык интерфейса
- **Category** — категория товаров
- **Product** — товар каталога
- **Cart** — позиции в корзине
- **Order** — заказ пользователя
- **OrderItem** — состав заказа на момент оформления

## Установка

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/0xPrecision/Demo_Shop_Bot.git
cd Demo_Shop_Bot
```

### 2. Создайте виртуальное окружение

```bash
python -m venv .venv
source .venv/bin/activate
```

### 3. Установите зависимости

```bash
pip install -r requirements.txt
```

### 4. Создайте `.env`

Минимальный пример:

```env
TELEGRAM_BOT_TOKEN=your_bot_token
ADMIN_IDS=123456789,987654321
```

### 5. Запустите Redis

По умолчанию проект ожидает Redis по адресу:

```text
redis://localhost:6379/0
```

### 6. Запустите проект

```bash
python main.py
```

## База данных и миграции

По умолчанию проект использует SQLite-базу `shop.db`.

Если база уже существует, перед применением миграций создаётся резервная копия в каталоге `backups/`.

### Применение миграций

```bash
aerich upgrade
```

### Инициализация миграций

```bash
aerich init-db
```

## Как работает бот

### Пользовательский сценарий

1. Пользователь открывает бот и переходит в каталог
2. Просматривает карточки товаров и добавляет нужные позиции в корзину
3. Переходит к оформлению заказа
4. Вводит или подтверждает контактные данные
5. Выбирает способ получения заказа
6. Подтверждает заказ
7. После оформления может просматривать историю своих заказов

### Административный сценарий

1. Администратор открывает административное меню
2. Управляет категориями и товарами
3. Просматривает заказы
4. Обновляет статусы заказов
5. Выполняет поиск по товарам и заказам
6. Просматривает статистику
7. Экспортирует заказы в CSV

## Локализация

Проект поддерживает два языка интерфейса:
- **русский**
- **английский**

Переводы хранятся в JSON-файлах внутри `services/locales/` и подключаются через middleware локализации.
